Class: BrowserAction

(abstract) Scenarii~BrowserAction()

The BrowserAction is the class you need to override to describe the behavior of you scenarii element (type action) on the browser side. A scenarii action element will be instantiated through this extending class on the browser side.

In your extending class, you MUST override these attributes:

  • .type = { name: 'MyOwnActionType', shortLabel: 'My action', fullLabel: 'My action that does something' }
  • .get name()
  • .get shortLabel()
  • .get fullLabel()

You can also override these attributes (optional):

  • .constructor()
  • .get EditForm()

You need also to override ServerAction in another class to describe the server side behavior of your action.

Implementation example: see below.

Constructor

(abstract) new BrowserAction()

You can override the constructor. Don't forget to pass data parameter to the super() constructor.

EditForm

Must return the component to use as Setting paneml to set/update the action instance. If null, there is no setting panel, the action instance is created directly.

fullLabel

Returns the long label of the action instance (not the action type). The full label must be concise but fully qualified (limit to a line of description).

eg.: "Will turn ON the switch 'Room light #2'"

name

Returns the name of the action instance (not the action type). The name must be very short (limit to ~25 chars approximately). It is often the name of the element that is manipulated through the action.

eg.: "Room light #2"

shortLabel

Returns the short label of the action instance (not the action type). The short label must be quite short (limit to ~40 chars approximately). It is often the short summary of the action.

eg.: "'Room light #2' > ON"

type

Properties to override in each extending class. Type properties are supposed to describe the action type itself (not the instances of that action type).

Properties:
Name Type Description
name string

A unique identifier for your class. Often the class name. Used in logs and for technical reasons.
shortLabel string

A SHORT description of your action type. Try to limit to ~25 chars approximately.
fullLabel string

A full description of your action type. Will complete the short label, so don't need to repeat the shortLabel inside the fullLabel, just complete. Limit to 2 lines of description.
