Class: ServerState

(abstract) Scenarii~ServerState()

The ServerState is the class you need to override to describe the behavior of you scenarii element (type state) on the server side. A scenarii state element will be instantiated through this extending class on the server side.

In your extending class, you MUST override these attributes:

  • .get name()

You can also override these attributes (optional):

  • .constructor()
  • .get state()
  • .set state()
  • .preValidate()

You need also to override BrowserAction in another class to describe the browser side behavior of your action.

Implementation example: see below.

Constructor

(abstract) new ServerState()

You can override the constructor. Don't forget to pass data parameter to the super() constructor.

Members

name

Returns the name of the state instance (not the state type). The name must be very short (limit to ~25 chars approximately). It is often the name of the concept that is manipulated through the state.

eg.: "Alarm level"

Methods

preValidate(state, oldState) → {boolean}

Prevalidate the new state value. This call will ask state listeners with the new value, to know if one of the listeners accepts the new value. If one reject the new value, then the state is not updated.

Parameters:
Name Type Description
state *

The new value proposed for the state.
oldState *

The value before update, to let listener compare and decide.
