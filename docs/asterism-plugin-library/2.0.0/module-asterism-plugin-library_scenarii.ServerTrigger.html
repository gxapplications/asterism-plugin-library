Class: ServerTrigger

(abstract) asterism-plugin-library/scenarii.ServerTrigger()

The ServerTrigger is the class you need to override to describe the behavior of you scenarii element (type trigger) on the server side. A scenarii trigger element will be instantiated through this extending class on the server side.

In your extending class, you MUST override these attributes:

  • .get name()
  • .reschedule()

You can also override these attributes (optional):

  • .constructor()

You need also to override BrowserTrigger in another class to describe the browser side behavior of your trigger.

Implementation example: see below.

Constructor

(abstract) new ServerTrigger()

You can override the constructor. Don't forget to pass data parameter to the super() constructor.

Members

name

Returns the name of the trigger instance (not the trigger type). The name must be very short (limit to ~25 chars approximately). It is often the name of the element that is looked through the trigger.

eg.: "Room light sensor"
