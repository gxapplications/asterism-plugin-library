Class: ConditionsDropdown

asterism-plugin-library/scenarii.ConditionsDropdown

This React component shows a dropdown with available scenarii elements of 'condition' type. This is use to choose an existing condition (that is public, so not inside another scenarii element like a procedure) or to create a new one on the fly (optional). You can filter conditions you want with a type filter as a callback, select default element, etc...

Example

Example of an ConditionsDropdown. Use it in React JSX syntax

<ConditionsDropdown defaultConditionId={this.state.conditionId} onChange={this.conditionChanged.bind(this)}
  theme={theme} animationLevel={animationLevel} services={services} />

The asterism main parameter for visual animations. Often available from the parent component, or in the mainState object of a context.
defaultConditionId string

The condition ID to pre-select at initialization.
onChange function

A callback when the user made a choice (in case of "new condition" choice, called only after the element creation).
dropdownId string

An ID, just in order to be able to make a dom query on it...
parentIdForNewInstance string

Optional parent ID element for creation of a "new condition" case. In that case, the new element will not appear in the main condition list! None by default.
label string

The label to put above the input, in the user interface.
icon string

The icon to put on the left of the input, in the user interface.
noCreationPanel boolean

True to avoid opening a popin (a modal frame with settings of the new element). False by default.
typeFilter function

A filter to choose what type of new conditions can be created from the dropdown.
instanceFilter function

A filter to choose what existing public conditions should be listed in the dropdown.
